# 호러 아이디어

이 문서는 `escape from the office`의 공포 연출 아이디어 저장소다.
공식 규칙은 `docs/design/UI_Rules.md`를 우선한다.
이 문서의 아이디어는 인카운터, 로그, ASCII 연출, 사내망 콘텐츠로 옮기기 전의 원천 노트로 사용한다.

## 핵심 감각

> 터미널 자체가 믿을 수 없는 공간이 된다.

플레이어가 반복해서 이런 생각을 하게 만드는 것이 목표다.

> 내가 잘못 본 건가?

초반에는 웃기고 이상한 회사 괴담처럼 시작한다.
중반부터 사내망, 로그, 공간, 선택지가 조금씩 틀어진다.
후반에는 회사 시스템이 현실을 덮어쓰는 듯한 감각으로 확장한다.

## 톤 진행

```text
초반: 이상한 회사 괴담, 블랙코미디
중반: 사내망과 공간 오류, 미스터리
후반: 격리된 차원, 현실 붕괴, 코스믹 호러
```

## 사내 시스템 공포

### 근태 시스템

```text
[근태 시스템]
현재 상태: 근무 중
퇴근 승인: 대기 중
```

후속 변형:

```text
[근태 시스템]
현재 상태: 근무 중
퇴근 승인: 반려됨
반려 사유: 아직 회의에 참석하지 않음
```

### 시설팀 공지

```text
[시설팀 공지]
13층에서 자기 자신을 목격하신 경우,
눈을 마주치지 마시기 바랍니다.
```

### 사내망 상태

```text
[NETWORK QUALITY: GOOD]
[NETWORK QUALITY: POOR]
[CONNECTION TO REALITY UNSTABLE]
```

### 회의실 예약 패널

```text
[긴급] 전 직원 참석
장소: 회의실 B
시간: 23:17
주최: 인사팀
```

마지막 문장:

```text
이 회사에는 인사팀이 없다.
```

## ASCII/화면 연출

### 미세한 프레임 변화

ASCII 아트는 대놓고 변하지 않는다.
플레이어가 확신하지 못할 정도로만 바뀌고 다시 정상으로 돌아온다.

예:

```text
│  │  │
```

잠깐:

```text
│╱ │  │
```

원칙:

> “방금 바뀐 것 같은데?”가 “괴물이 나타났다”보다 강하다.

### 엘리베이터 프레임 후보

```text
assets/ascii/elevator_normal.txt
assets/ascii/elevator_shadow_1.txt
assets/ascii/elevator_shadow_2.txt
assets/ascii/elevator_hand.txt
```

연출:

- 문이 천천히 열림
- 층수 표시가 7F와 13F 사이에서 흔들림
- 현재 인원 수가 1과 2를 반복
- 선택지 `엘리베이터에 탄다`가 `이미 탔다`로 바뀜

## 로그 기반 공포

### 미래 로그

```text
/var/log/survivors/

park_2026-05-15.log
kim_2026-05-15.log
lee_2026-05-15.log
you_2026-05-15.log
you_2026-05-16.log
```

```text
[2026-05-16 03:12:44]
나는 엘리베이터를 타지 말았어야 했다.

[2026-05-16 03:12:45]
그런데 이미 탔다.
```

### 사망 후 로그

```text
[LOG] 사용자가 응답하지 않습니다.
[LOG] 사용자가 계속 입력하고 있습니다.
```

### 자기 이름 로그

플레이어가 아직 하지 않은 행동이 본인 이름으로 기록되어 있다.

```text
[23:19:04] user accepted meeting invite.
[23:19:07] user entered conference_room_b.
[23:19:08] user has not left.
```

## 선택지 오염 샘플

기본:

```text
[1] 엘리베이터에 탄다
[2] 비상계단으로 간다
[3] 돌아간다
```

변형:

```text
[1] 이미 탔다
[2] 비상계단으로 간다
[3] 돌아간다
```

또는:

```text
[1] 엘리베이터에 탄다
[2] 비상계단으로 간다
[3] 돌아간다
[4] 엘리베이터가 당신을 탄다
```

입력 왜곡 샘플:

```text
> 2
입력 확인:
> 엘리베이터로 이동한다
```

이 연출은 `docs/design/UI_Rules.md`의 제한 조건을 만족하는 장면에서만 사용한다.

## 장소별 아이디어

### 내 자리 / 개발팀 사무실

- 빌드는 끝나지 않았는데 로그 시간이 미래로 찍힘
- 마지막 commit message가 현재 상황을 암시함
- 사내 메신저에 사라진 팀원의 메시지가 옴
- 모니터는 켜져 있는데 의자는 너무 반듯함

### 탕비실

- 커피머신이 “정상 추출 중”만 반복
- 정수기 물소리가 들리지만 이 층에는 정수기가 없음
- 냉장고 자석 아래 현실 연결 힌트
- 유통기한이 내일이 아니라 어제로 계속 바뀜

### 회의실

- 존재하지 않는 회의 참석자
- 플레이어가 참석자로 등록되어 있음
- 회의록에 아직 하지 않은 발언이 적힘
- 회의가 끝나지 않으면 문이 열리지 않는 것처럼 보임

### 복합기 구역

- 전원이 꺼져 있는데 출력물이 나옴
- 프린트 대기열에 플레이어 이름이 있음
- 마지막 장만 출력되지 않음
- 출력 여백이 현실 연결 힌트가 됨

### 비상계단

- 7층에서 한 층 내려갔는데 다시 7층
- 층수 표기가 빠짐
- 13층 경고 공지
- 발소리가 위에서 아래로가 아니라 뒤에서 들림

### 서버실 앞

- 출입 권한이 확인되었지만 문은 열리지 않음
- 보안 패널이 플레이어를 이미 입실 처리함
- 차가운 바람이 문 아래가 아니라 화면에서 느껴지는 것처럼 묘사됨
- 사내망 로그에서 격리 현상 첫 명확한 단서
