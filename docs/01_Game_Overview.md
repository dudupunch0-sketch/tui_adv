# escape from the office 게임 개요

## 한 줄 설명

`escape from the office`는 갑자기 사람 전부가 사라지고 공간 또는 차원 단위로 격리된 대기업 연구개발동에서, 플레이어가 체력/정신력/배터리/허기/갈증을 관리하며 탈출, 정복, 진실 발견, 현실 연결 히든 루트를 추구하는 TUI 랜덤 인카운터 선택지 생존 게임이다.

## 핵심 판타지

플레이어는 너무 익숙해서 무심코 지나쳤던 회사 공간을 다시 읽어야 한다.

- 사내망 알림은 끊겼다가 다시 살아난다.
- 인터넷은 제한되지만, 누가 보냈는지 알 수 없는 사내 메시지는 도착한다.
- 회의실 예약 패널은 존재하지 않는 회의를 표시한다.
- 복합기는 꺼져 있는데 출력물을 뱉는다.
- 탕비실의 물과 음식은 생존 수단이면서 이상현상의 단서다.
- 비상계단은 탈출구일 수도 있고, 같은 층으로 되돌아오는 반복 공간일 수도 있다.

## 배경

배경은 실제 특정 회사명을 직접 쓰지 않는, “국내 최고 대기업 IT/반도체 회사의 연구개발동” 같은 추상화된 사무실이다.
서버실 존재 여부는 현실 고증보다 게임 기능을 우선한다. 필요한 경우 “서버실 앞”, “장비실”, “보안구역”, “전산실” 같은 게임용 구역으로 처리한다.

## 1차 재난 타입

1차 재난 타입은 “불명 재난”이다.

상황:

- 갑자기 모든 사람이 보이지 않는다.
- 회사 건물 또는 연구개발동 일부가 공간/차원 단위로 격리된 느낌이다.
- 외부 인터넷은 제한된다.
- 사내망, 메신저, 예약 패널, 복합기, 보안 시스템은 간헐적으로 작동한다.
- 간혹 연락이 오지만, 상대가 사람인지 시스템인지 과거의 기록인지 확실하지 않다.
- 재난의 정체는 게임 진행 중 단서로 추론한다.

## 톤

기본 톤은 “블랙코미디 회사 괴담 + 코스믹 호러”다.

초반:

- 야근, 사내망 장애, 회의실 예약, 탕비실, 복합기 같은 현실적인 회사 소재로 시작한다.
- 블랙코미디와 회사 생활 풍자가 섞인다.

중후반:

- 회사의 규칙이 물리 법칙처럼 작동한다.
- 사내문서와 회의록이 현실을 덮어쓴다.
- 사람의 부재보다 “회사는 여전히 정상 운영 중”이라는 사실이 더 무섭게 느껴진다.

## 핵심 자원

| 자원 | 방향 | 역할 |
|---|---|---|
| 체력 | 높을수록 좋음 | 물리적 생존력 |
| 정신력 | 높을수록 좋음 | 괴현상과 회사적 광기를 버티는 힘 |
| 배터리 | 높을수록 좋음 | 사내망, 조명, 녹음, 촬영, 지도 확인에 쓰는 정보 접근 자원 |
| 허기 | 낮을수록 좋음 | 시간이 지날수록 쌓이는 생존 압박 |
| 갈증 | 낮을수록 좋음 | 허기보다 빠르게 위험해지는 탈수 압박 |

## 1차 수직 슬라이스 범위

### 위치 8개

- 내 자리
- 개발팀 사무실
- 복도
- 탕비실
- 회의실
- 복합기 구역
- 서버실 앞
- 비상계단

### 아이템 10개

- 생수
- 커피
- 과자
- 컵라면
- 구급상자
- 보조배터리
- 손전등
- 사원증
- 구겨진 출력물
- 퇴사자의 메모

### 주요 인카운터

- 퇴사자의 메신저
- 복합기가 혼자 출력한다
- 회의실 예약 패널
- 탕비실 커피머신
- 물품창고 비상 보급함
- 정수기의 이상한 물
- 서버실 앞의 차가운 바람
- 비상계단의 발소리
- 존재하지 않는 층의 엘리베이터
- 사내 방송
- 회의록에 적힌 내 이름
- 냉장고 안의 쪽지
- 책상 아래 손전등
- 보안 카메라의 시선
- 옥상의 제한된 외부 신호
- 지하주차장의 시동음

### 주요 엔딩

- 실패 엔딩: 체력/정신력/허기/갈증 한계
- 탈출 엔딩: 비상계단을 통한 탈출
- 탈출 엔딩: 옥상 외부 신호 송신
- 히든 현실 연결 엔딩: 첫 번째 현실 메모의 중간 수준 힌트 획득

## 현실 연결 강도

현실 연결은 “중간 연결”을 기본으로 한다.

게임은 다음 수준까지 안내한다.

- 구역 힌트
- 사물 힌트
- 행동 힌트

예:

- “커피 냄새가 남아 있는 방.”
- “차가운 문 옆, 작은 자석.”
- “붙어 있는 것 아래를 보라.”

공개 문서나 공개 데이터에는 실제 최종 위치를 직접 쓰지 않는다.
최종 위치는 로컬 비공개 파일에서만 관리한다.

## 주요 엔딩 방향

### 탈출

회사의 물리적 또는 차원적 경계를 찾아 벗어난다.

### 정복

보안실, 사내망, 서버/전산 구역, 사내 방송 시스템을 장악해 회사의 규칙을 역이용한다.

### 진실

사람들이 왜 사라졌는지, 회사 공간이 왜 격리되었는지, 사내망이 누구와 연결되는지 밝혀낸다.

### 히든 현실 연결

게임 속 비밀을 풀어 현실 사무실 어딘가에 숨겨진 개발자의 메모 또는 작은 보상으로 이어지는 힌트를 얻는다.

## 설계 원칙

- 1차 구현은 불명 재난 하나에 집중한다.
- 모든 콘텐츠는 나중에 데이터 파일로 분리할 수 있게 설계한다.
- 게임 엔진과 TUI 표현을 분리한다.
- 회사 소재는 풍자적으로 쓰되, 실제 회사명/직원명/민감정보는 쓰지 않는다.
- 현실 연결 힌트는 안전한 공용 공간만 대상으로 한다.
