# 맵 설계

## 목적

맵은 실제 사무실을 직접 복제하지 않고, “국내 최고 대기업 IT/반도체 회사의 연구개발동” 같은 공간감을 추상화한다.
실제 서버실 존재 여부나 정확한 배치는 몰라도 된다. 게임은 현실에서 익숙한 사무실/연구동의 기능적 구역을 사용한다.

## 맵 설계 원칙

1. 실제 회사명, 층수, 보안 구조, 내부 배치 같은 민감정보는 쓰지 않는다.
2. 플레이 가능한 구역은 기능과 분위기로 정의한다.
3. 각 위치는 생존, 정보, 위험, 루트 중 하나 이상의 역할을 가진다.
4. 1차 수직 슬라이스는 작은 맵으로 시작한다.
5. 나중에 실제 사무실 기반 힌트를 넣을 때도 공개 문서에는 중간 힌트까지만 둔다.

## 1차 위치 목록

| 위치 | 역할 | 주요 자원/루트 |
|---|---|---|
| 내 자리 | 시작 지점, 개인 단서 | 사내망, 퇴사자의 메모, 배터리 판단 |
| 개발팀 사무실 | 기본 허브 | 책상, 손전등, 메신저, 반복 알림 |
| 복도 | 이동 허브 | 위험도, 발소리, CCTV |
| 탕비실 | 생존 회복 | 생수, 커피, 과자, 컵라면, 현실 연결 힌트 |
| 회의실 | 정신력 위험/진실 단서 | 예약 패널, 회의록, 반복 회의 |
| 복합기 구역 | 히든 루트 시작 | 출력물, 토너, 사내망 프린트 로그 |
| 서버실 앞 | 정보/정복/진실 관문 | 사원증, 보안 패널, 차가운 바람 |
| 비상계단 | 탈출 루트 | 어둠, 발소리, 반복 층, 탈출 조건 |

## 위치 연결 초안

```text
             [회의실]
                |
[내 자리] - [개발팀 사무실] - [복도] - [비상계단]
                |              |
             [복합기 구역]     [서버실 앞]
                |
             [탕비실]
```

연결은 실제 회사 배치가 아니라 게임용 추상 구조다.
필요하면 나중에 실제 사무실의 큰 특징만 반영해 조정한다.

## 위치별 상세

### 내 자리

역할:

- 시작 지점
- 플레이어의 개인 맥락 제공
- 첫 사내망 알림 또는 퇴사자의 메시지 발생

주요 이벤트:

- 퇴사자의 메신저
- 모니터에 뜬 알 수 없는 코드
- 개인 서랍에서 발견한 보조배터리 또는 메모

주요 선택:

- 사내망 메시지를 확인한다
- 책상 서랍을 연다
- 개발팀 사무실로 나간다

### 개발팀 사무실

역할:

- 1차 맵의 중심 허브
- 회사 블랙코미디 분위기 제공
- 사무실 전체가 비었음을 체감하는 공간

주요 이벤트:

- 빈 좌석에서 울리는 키보드 소리
- 자동 빌드 실패 알림
- 책상 아래 손전등
- 누군가의 모니터에 떠 있는 “정상 근무 중” 표시

주요 선택:

- 주변 책상을 조사한다
- 복도로 이동한다
- 회의실로 이동한다
- 복합기 구역으로 이동한다

### 복도

역할:

- 이동 허브
- 위험도와 긴장감 상승
- CCTV/발소리/사내방송 이벤트 발생

주요 이벤트:

- 보안 카메라의 시선
- 사내 방송
- 비상등 깜빡임
- 아무도 없는데 지나가는 사원증 인식음

주요 선택:

- 비상계단으로 간다
- 서버실 앞으로 간다
- 개발팀 사무실로 돌아간다
- 소리를 피해 기다린다

### 탕비실

역할:

- 허기/갈증 회복
- 블랙코미디와 괴담이 만나는 공간
- 현실 연결 힌트의 주요 후보

주요 아이템:

- 생수
- 커피
- 과자
- 컵라면

주요 이벤트:

- 탕비실 커피머신
- 정수기의 이상한 물
- 냉장고 안의 쪽지
- 아무도 없는데 줄어드는 종이컵

현실 연결:

- 중간 수준 힌트까지 제공 가능
- 예: “커피 냄새가 남아 있는 방”, “차가운 문 옆”, “붙어 있는 것 아래”
- 실제 최종 위치는 공개 문서에 쓰지 않음

### 회의실

역할:

- 정신력 위험 구역
- 진실 루트 단서
- 코스믹 호러 분위기 강화

주요 이벤트:

- 회의실 예약 패널
- 회의록에 적힌 내 이름
- 존재하지 않는 부서의 전체회의
- 입장하면 늘 같은 시간이 되는 회의실

주요 선택:

- 회의에 참석한다
- 예약 패널을 확인한다
- 회의록을 읽는다
- 문을 잠그고 나간다

### 복합기 구역

역할:

- 히든 현실 연결 루트 시작점
- 사내망/프린트 로그 단서
- 회사 괴담에 잘 어울리는 물리 장치

주요 이벤트:

- 복합기가 혼자 출력한다
- 출력물에 아직 하지 않은 선택이 적혀 있다
- 토너 카트리지 뒤에 구겨진 출력물이 있다
- 프린트 대기열에 알 수 없는 문서가 있다

주요 선택:

- 출력물을 읽는다
- 출력물을 챙긴다
- 토너를 확인한다
- 프린트 로그를 추적한다

### 서버실 앞

역할:

- 정보, 정복, 진실 루트의 관문
- 실제 서버실이 있는지와 무관하게 “보안/전산 구역”으로 사용
- 1차 구현에서는 내부 진입 전 단계까지만 다룰 수 있음

주요 이벤트:

- 서버실 앞의 차가운 바람
- 사원증 인식 실패
- 보안 패널의 알 수 없는 안내문
- 안쪽에서 들리는 키보드 소리

주요 선택:

- 사원증을 찍는다
- 휴대폰으로 패널을 촬영한다
- 사내망 로그를 확인한다
- 돌아간다

### 비상계단

역할:

- 1차 탈출 루트
- 어둠, 발소리, 반복 층 연출
- 배터리/체력/정신력 압박

주요 이벤트:

- 비상계단의 발소리
- 같은 층으로 되돌아옴
- 문 너머에서 들리는 안내 방송
- 아래층 표시가 사라짐

주요 선택:

- 아래로 내려간다
- 위로 올라간다
- 손전등을 켠다
- 문을 막는다
- 개발팀 사무실로 돌아간다

## 확장 위치 후보

2차 이후 추가 후보:

- 서버실 내부 또는 전산실
- 보안실
- 대표실/임원실
- 엘리베이터
- 옥상
- 지하주차장
- 로비
- 화장실
- 물품창고
- 흡연구역 또는 휴게공간

## 맵과 루트 연결

### 탈출 루트

비상계단 중심.
나중에 옥상, 지하주차장, 로비 루트 추가 가능.

### 정복 루트

서버실 앞 → 서버실/전산실 → 보안실 → 사내 방송 또는 출입 시스템 장악.

### 진실 루트

내 자리 → 퇴사자의 메시지 → 회의실 예약 → 복합기 출력물 → 서버 로그.

### 현실 연결 루트

복합기 구역 → 구겨진 출력물 → 탕비실/회의실 단서 → 중간 수준 현실 힌트.

## TUI 표시 초안

초기에는 복잡한 지도보다 위치 목록과 연결 표시가 좋다.

예:

```text
현재 위치: 개발팀 사무실

이동 가능:
1. 내 자리
2. 복도
3. 회의실
4. 복합기 구역
```

나중에 ASCII 미니맵을 추가할 수 있다.
