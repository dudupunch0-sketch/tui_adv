# 현실 연결 설계

## 목적

현실 연결은 `escape from the office`의 히든 루트다.
플레이어가 게임 속 비밀을 파헤치면, 실제 사무실 어딘가에 개발자가 숨겨둔 메모 또는 작은 보상을 찾을 수 있는 힌트를 얻는다.

이 문서는 원칙과 표현 방식을 정의한다.
실제 최종 위치는 이 문서에 쓰지 않는다.

## 연결 강도

현재 확정된 강도는 “중간 연결”이다.

게임이 제공하는 것:

- 구역 힌트
- 사물 힌트
- 행동 힌트

게임이 공개 문서/공개 데이터에서 직접 제공하지 않는 것:

- 실제 회사명
- 실제 층수
- 실제 좌석 번호
- 실제 최종 위치 문장
- 개인 이름
- 보안상 민감한 위치

## 힌트 단계

### 1단계: 분위기 힌트

장소의 성격을 암시한다.

예:

- “커피 냄새가 남아 있는 곳.”
- “차갑고 조용한 문 근처.”
- “아무도 읽지 않는 안내문 아래.”

### 2단계: 구역 힌트

게임 속 구역과 현실 구역을 연결한다.

예:

- “탕비실과 닮은 곳.”
- “출력물이 쌓이는 곳.”
- “회의가 시작되기 전 모두가 지나치는 곳.”

### 3단계: 사물 힌트

특정 사물 유형을 가리킨다.

예:

- “자석.”
- “프린터 옆 종이.”
- “냉장고 문.”
- “게시판 모서리.”

### 4단계: 최종 위치

최종 위치는 로컬 비공개 파일에서만 관리한다.
공개 문서에는 쓰지 않는다.

## 안전 원칙

현실 연결 대상은 반드시 다음 조건을 만족해야 한다.

- 공용 공간이다.
- 허가 없이 열면 안 되는 장소가 아니다.
- 개인 책상이나 개인 물건이 아니다.
- 높은 곳, 전기설비, 위험 물질 근처가 아니다.
- 회사 보안/기밀/고객 정보와 무관하다.
- 주변 사람에게 불편함을 주지 않는다.
- 작은 메모나 스티커처럼 무해한 보상만 둔다.

## 금지 위치

사용하지 않는다.

- 개인 책상 서랍
- 사물함
- 잠긴 회의실/서버실/보안실
- 전기실/기계실/위험 설비 근처
- 천장, 배선함, 소화설비 내부
- 보안 문서가 있는 곳
- 실제 회사 운영에 방해되는 장비 위/안

## 공개/비공개 파일 경계

공개 가능:

- 힌트 스타일
- 힌트 단계
- 예시용 가짜 위치
- 게임 내 플래그 이름
- 안전 원칙

비공개 유지:

- 실제 최종 위치
- 실제 보상 배치 사진
- 실제 층/동/좌석 번호
- 실제 직원/팀 이름
- 실제 보안 구조

권장 파일:

```text
docs/story/Reality_Link.md          # 이 문서. 공개 가능 원칙만 기록
src/tui_adv/data/secrets.example.*  # 예시 데이터. 실제 위치 없음
private/Real_Office_Secrets.local.md # 실제 위치. git ignore 필요
private/secrets.local.*             # 실제 게임 로컬 데이터. git ignore 필요
```

## 게임 내 표현 예시

### 복합기 루트

```text
복합기가 마지막 장을 출력한다.

문서 제목은 없다.
페이지 하단에는 사내 양식과 다른 글씨가 있다.

커피 냄새가 남아 있는 방.
차가운 문 옆.
붙어 있는 것 아래.

이 문장은 게임 속 탕비실만을 말하는 것 같지 않다.
```

### 회의실 루트

```text
회의록 마지막 줄에 참석자 명단 대신 장소가 적혀 있다.

모두가 들어오지만 아무도 오래 머물지 않는 곳.
예약되지 않았지만 항상 쓰이는 곳.
작은 표시 아래.
```

### 사내망 루트

```text
사내망 로그는 중간에 깨져 있다.
하지만 반복되는 문자열 하나가 눈에 들어온다.

PANTRY / MAGNET / BELOW

누군가 일부러 너무 직접적인 단어만 남긴 것 같다.
```

## 구현 방향

현실 연결은 일반 엔딩과 다르게 처리한다.

- 게임 내에서는 힌트 단계만 unlock한다.
- 실제 최종 위치는 로컬 secret 데이터가 있을 때만 표시한다.
- 로컬 secret 데이터가 없으면 중간 힌트까지만 보여준다.
- 공개 배포판에서는 현실 보물 찾기 대신 “개발자의 메모가 어딘가에 있다”는 메타 엔딩으로 마무리할 수 있다.

## 체크리스트

새 현실 연결 힌트를 추가할 때 확인한다.

- [ ] 공용 공간인가?
- [ ] 위험하지 않은가?
- [ ] 개인 물건을 건드리지 않는가?
- [ ] 회사 보안과 무관한가?
- [ ] 공개 데이터에 실제 위치가 들어가지 않았는가?
- [ ] 힌트가 너무 직접적이지 않은가?
- [ ] 플레이어가 자연스럽게 추론할 수 있는가?
