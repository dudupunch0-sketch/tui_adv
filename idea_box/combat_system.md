# 전투 시스템

## 문서 목적

이 문서는 **Escape from the Office**의 전투 시스템 방향을 정리한 기획 문서다.

본 전투 시스템은 전통적인 턴제 RPG나 액션 게임이 아니라, **실시간 자동 난투 + 상황 개입** 구조를 기반으로 한다.

플레이어는 모든 공격을 직접 조작하지 않는다. 대신 캐릭터들이 자동으로 뒤엉켜 싸우는 과정에서, 중요한 순간에만 선택지를 통해 개입한다.

---

# 1. 핵심 정의

## 한 줄 정의

> 실시간으로 자동 진행되는 MMA식 난투 속에서, 플레이어가 결정적인 상황에만 개입해 전투의 흐름을 바꾸는 시스템.

## 핵심 목표

- 싸우는 느낌 유지
- 조작 피로 감소
- 상황 판단 재미 제공
- 세계관 범용성 확보
- 유쾌한 분위기 형성
- 전투를 짧은 드라마처럼 구성

---

# 2. 전투의 기본 방향

이 전투는 다음 요소를 중심으로 하지 않는다.

- 검술 콤보
- 마법 연계
- 스킬 쿨타임 회전
- 정밀 액션 조작
- HP 숫자 깎기

대신 전투의 중심은 다음에 있다.

- 거리
- 균형
- 압박
- 호흡
- 자세
- 시야
- 공포
- 분노
- 무기 제어
- 환경 활용

즉, 전투는 수치 교환이 아니라 **흐름 싸움**이다.

---

# 3. 전투의 핵심 감정

이 시스템이 노리는 감정은 화려함보다 **아슬아슬하고 웃긴 현실적 난투**에 가깝다.

전투는 멋진 액션보다 다음 감각을 우선한다.

- 계획대로 흘러가지 않는 싸움
- 서로 밀치고 붙잡고 넘어지는 몸싸움
- 예상치 못한 사고
- 환경에 휘말리는 난장판
- 중요한 순간에 흐름을 바꾸는 판단

핵심 톤은 다음과 같다.

> 세계관은 진지한데, 싸움은 이상하게 현실적이다.

예를 들어 판타지 세계에서도 드래곤 목에 매달리면 초크를 걸 수 있고, 회사 배경에서도 일그러진 부장을 복사기 쪽으로 밀어붙일 수 있으며, 무협 세계에서도 자세가 흐트러진 고수에게 태클을 걸 수 있다.

---

# 4. 기본 전투 루프

전투는 다음 흐름으로 진행된다.

```text
전투 시작
→ 전투 전 전투력 비교
→ 전투 등급 및 개입 예산 결정
→ 자동 교전 시작
→ 상황 발생
→ 필요 시 플레이어 개입
→ 상태 변화
→ 자동 교전 지속
→ 결착
```

플레이어는 계속 입력하지 않는다. 전투의 대부분은 자동으로 진행되며, 플레이어는 제한된 개입 기회에서 전투의 방향을 바꾼다.

---

# 5. 자동 전투

## 자동 전투의 역할

자동 전투는 단순히 공격 모션을 반복하는 것이 아니다.

이 시스템에서 자동 전투는 다음 역할을 가진다.

> 스포츠 중계 + 물리 코미디 + 상황극 생성기

즉, 자동 전투는 플레이어가 개입할 만한 상황을 만들어내는 기반이다.

## 자동 행동 예시

캐릭터들은 실시간으로 다음 행동을 자동 수행한다.

- 주먹 공격
- 발차기
- 밀치기
- 붙잡기
- 클린치
- 태클 시도
- 회피
- 거리 조절
- 벽으로 몰기
- 넘어짐
- 다시 일어나기
- 숨 고르기
- 무기 휘두르기
- 무기 놓치기
- 동료 지원
- 패닉 도주
- 환경에 걸려 휘청임

중요한 것은 전투 결과뿐 아니라 **싸우는 과정 자체가 재미있어야 한다**는 점이다.

---

# 6. 플레이어의 역할

플레이어는 전투자를 직접 조종하는 사람이 아니다.

플레이어의 역할은 다음에 가깝다.

- 감독
- 상황 판단자
- 흐름 조율자
- 결정적 순간에 개입하는 사람

플레이어는 다음을 판단한다.

- 지금 밀어붙일 것인가?
- 뒤로 빠질 것인가?
- 동료를 구할 것인가?
- 무기를 유지할 것인가?
- 무기를 버리고 붙잡을 것인가?
- 환경을 이용할 것인가?
- 위험하지만 큰 기회를 노릴 것인가?

따라서 전투의 재미는 조작 피지컬보다 **상황 판단과 흐름 장악**에서 나온다.

---

# 7. 개입 예산 시스템

## 핵심 원칙

전투 하나에서 플레이어에게 요구되는 개입 횟수는 제한된다.

> 전투 하나당 플레이어 개입 요구는 0~3회로 제한한다.

이는 전투가 선택지 반복 게임으로 변질되는 것을 막기 위한 핵심 규칙이다.

본 시스템의 중심은 자동 난투이며, 개입은 어디까지나 결정적 순간에만 발생해야 한다.

---

## 전투 시작 전 판정

전투가 시작되면 시스템은 먼저 다음을 비교한다.

```text
현재 플레이어 전투력
VS
몹의 전투력
```

이 비교는 전투 전에 이미 결정된다.

플레이어 전투력은 단순 레벨이나 공격력만 보지 않는다. 다음 요소를 종합한다.

- 현재 스탯
- 장비
- 무기 상태
- 체력 및 부상
- 호흡 상태
- 동료 수
- 동료 상태
- 사전 단서 보유 여부
- 현재 위치
- 환경 우위
- 적과의 거리
- 기습 여부
- 전투 직전의 심리 상태

몹의 전투력도 다음 요소를 고려한다.

- 기본 전투 능력
- 몹 등급
- 무기 보유 여부
- 특수 능력
- 숫자 우위
- 현재 상태
- 지형 적응도
- 공포 또는 분노 상태
- 보스 패턴 여부

이 비교 결과에 따라 전투 등급과 개입 예산이 결정된다.

---

## 전투 등급별 개입 횟수

| 전투 등급 | 기준 | 개입 요구 횟수 |
|---|---|---:|
| 약한 상대 | 플레이어가 명확히 우세한 전투 | 0회 |
| 평범한 상대 | 큰 위험은 없지만 변수가 있는 전투 | 0~1회 |
| 강한 정예몹 | 플레이어와 비슷하거나 일부 우세한 상대 | 1~2회 |
| 강한 보스몹 | 전투의 핵심 장면이 되는 상대 | 최대 3회 |

보스전이라도 개입은 최대 3회를 넘지 않는다.

---

## 개입 예산의 의미

개입 예산은 전투 중 선택지가 등장할 수 있는 최대 횟수다.

예를 들어 개입 예산이 2라면, 해당 전투에서 플레이어가 선택지를 통해 전투에 개입하는 횟수는 최대 2회다.

다만 개입 예산이 있다고 해서 반드시 전부 사용되는 것은 아니다.

예시:

```text
정예몹 전투
개입 예산: 2회

실제 진행:
1차 개입 발생
자동 전투에서 플레이어가 유리한 흐름 확보
추가 위기 없음
전투 종료

실제 개입 횟수: 1회
```

즉, 개입 예산은 **최대치**이며, 실제 개입 횟수는 전투 흐름에 따라 줄어들 수 있다.

---

## 등급별 전투 체감

### 약한 상대

약한 상대는 플레이어가 개입하지 않아도 자동으로 처리된다.

```text
전투 시작
→ 자동 난투
→ 제압
→ 종료
```

약한 상대에게 선택지가 자주 뜨면 전투 피로도가 증가하므로, 기본적으로 개입 요구는 없다.

---

### 평범한 상대

평범한 상대는 대부분 자동으로 처리되지만, 상황에 따라 1회 정도의 개입이 발생할 수 있다.

```text
전투 시작
→ 자동 난투
→ 상대가 의자를 집어 듦
→ 플레이어 개입
→ 흐름 변화
→ 종료
```

평범한 상대의 개입은 전투의 긴장감을 살리는 정도로만 사용한다.

---

### 강한 정예몹

정예몹은 전투 중 1~2회의 의미 있는 선택지를 제공한다.

```text
전투 시작
→ 자동 난투
→ 정예몹이 플레이어를 벽으로 몰아붙임
→ 1차 개입
→ 전세 변화
→ 클린치 상황 발생
→ 2차 개입 가능
→ 결착
```

정예몹 전투는 짧지만 기억에 남는 장면을 만드는 것이 목표다.

---

### 강한 보스몹

보스전은 최대 3회의 개입을 허용한다.

```text
전투 시작
→ 자동 난투
→ 1차 위기 또는 기회
→ 1차 개입
→ 보스 패턴 변화
→ 2차 위기 또는 기회
→ 2차 개입
→ 결착 직전 상황
→ 3차 개입
→ 전투 종료
```

보스전은 긴 조작전이 아니라, 2~3개의 강한 장면으로 구성된 짧은 전투 드라마에 가깝다.

---

# 8. 개입 발생 조건

개입은 아무 때나 발생하지 않는다.

개입은 반드시 **결정적 상황**에서만 발생한다.

## 결정적 상황 예시

- 상대의 균형이 크게 무너짐
- 플레이어가 벽에 몰림
- 동료가 붙잡힘
- 무기를 떨어뜨림
- 적이 무리하게 돌진함
- 환경 사물을 사용할 수 있음
- 적의 약점이 노출됨
- 보스가 큰 공격을 준비함
- 플레이어가 넘어짐
- 누군가 클린치 상태에 들어감
- 좁은 공간에서 무기가 걸림
- 사전 단서와 연결되는 순간이 발생함

선택지는 이 결정적 상황에 대한 대응으로만 등장한다.

---

## 개입의 설계 원칙

- 개입은 많을수록 좋은 것이 아니다.
- 개입은 전투의 흐름을 바꾸는 순간이어야 한다.
- 개입 선택지는 단순 공격 선택이 아니어야 한다.
- 선택 후 상태 변화가 즉시 체감되어야 한다.
- 약한 전투에는 개입을 거의 주지 않는다.
- 보스전도 최대 3회를 넘기지 않는다.

---

# 9. 상황 개입 시스템

## 기본 구조

전투 중 특정 조건이 발생하면 선택지가 등장한다.

예시:

```text
상대가 균형을 잃었다!

1. 밀어버린다
2. 다리를 노린다
3. 뒤로 빠진다
```

선택 후 다음이 발생한다.

- 자동 연출 실행
- 상태 변화 발생
- 위치 변화 발생 가능
- 동료 행동 변화 가능
- 적 행동 패턴 변화 가능
- 다시 자동 전투 지속

---

## 개입 선택지의 성격

선택지는 단순히 공격 종류를 고르는 것이 아니다.

좋은 선택지는 다음 중 하나 이상의 변화를 만든다.

- 거리 변화
- 균형 변화
- 압박 변화
- 호흡 변화
- 무기 제어 변화
- 동료 상태 변화
- 적의 심리 변화
- 환경 상태 변화
- 전투 종료 가능성 변화

예시:

```text
부장이 회의실 벽에 등을 부딪히며 휘청인다.

1. 어깨로 밀어붙인다
   → 압박 증가, 균형 추가 감소

2. 손목을 잡아 스테이플러를 떨어뜨린다
   → 적 무기 제어 감소

3. 뒤로 빠져 숨을 고른다
   → 플레이어 호흡 회복, 거리 증가
```

---

# 10. 핵심 상태 시스템

HP는 존재할 수 있지만 중심이 아니다.

전투의 핵심은 상태 변화다.

| 상태 | 의미 |
|---|---|
| 거리 | 서로 얼마나 가까운가 |
| 균형 | 넘어질 위험 |
| 압박 | 벽, 구석, 다수에게 몰린 정도 |
| 호흡 | 지침, 체력 소모, 반응 저하 |
| 자세 | 공격, 방어, 회피 가능 상태 |
| 공포 | 위축, 도주, 패닉 가능성 |
| 분노 | 무리한 돌진, 판단 저하 |
| 시야 | 상대와 환경을 인식하는 정도 |
| 무기 제어 | 무기를 제대로 쓰고 있는 정도 |
| 동료 안정 | 동료가 전투를 유지할 수 있는 정도 |

전투는 상대의 HP를 깎는 것보다, 상대의 균형과 흐름을 무너뜨리는 것이 중요하다.

---

# 11. 상황 트리거 시스템

선택지는 랜덤으로 생성되지 않는다.

현재 전투 상황을 나타내는 태그를 기반으로 생성된다.

| 상황 태그 | 설명 |
|---|---|
| 균형 붕괴 | 자세가 무너짐 |
| 벽 근처 | 벽으로 몰거나 박을 수 있음 |
| 클린치 | 서로 붙잡고 있음 |
| 다운 | 누군가 넘어짐 |
| 동료 위험 | 동료가 붙잡히거나 포위됨 |
| 무기 이탈 | 무기를 떨어뜨림 |
| 환경 접근 | 책상, 의자, 복사기, 횃불 등을 활용 가능 |
| 분노 상태 | 상대가 무리하게 돌진하려 함 |
| 공포 상태 | 상대가 위축되어 있음 |
| 시야 차단 | 어둠, 연기, 조명, 군중 등으로 시야가 나쁨 |
| 좁은 공간 | 긴 무기나 다수전에 제약 발생 |
| 높은 곳 | 추락, 난간, 계단 활용 가능 |

예시:

```text
상대가 벽 근처에서 균형을 잃었다!

1. 어깨로 밀어 벽에 박는다
2. 다리를 걸어 넘어뜨린다
3. 거리를 벌리고 숨을 고른다
```

---

# 12. 선택지 생성 재료

선택지는 다음 세 요소를 조합해 생성된다.

## 12.1 현재 상황 태그

예시:

- 벽
- 책상
- 의자
- 복사기
- 난간
- 계단
- 유리창
- 좁은 복도
- 진흙
- 횃불
- 절벽
- 자동문
- 케이블

## 12.2 캐릭터 스탯

| 스탯 | 영향 |
|---|---|
| 체격 | 밀치기, 압박, 버티기 |
| 민첩 | 회피, 파고들기, 빠져나오기 |
| 관찰 | 약점 발견, 환경 활용 |
| 침착 | 패닉 감소, 위기 판단 |
| 공감 | 동료 구조, 협동 행동 |
| 투지 | 불리한 상황에서 버티기 |
| 비열함 | 눈 찌르기, 급소 공격, 속임수 |
| 숙련 | 무기 유지, 자세 회복 |

## 12.3 사전 단서

탐색이나 대화에서 얻은 정보가 전투 선택지를 바꾼다.

예시:

- 상대는 왼쪽 다리가 다쳤다.
- 상대는 빛에 민감하다.
- 상대는 겁이 많다.
- 상대는 권위에 약하다.
- 상대는 오래 움직이면 숨이 빨리 찬다.
- 상대는 한쪽 눈이 잘 보이지 않는다.
- 상대는 비싼 물건을 지나치게 아낀다.

예시:

```text
상대가 왼쪽 다리를 끌고 있다!

1. 왼쪽 다리를 노린다
2. 일부러 오른쪽으로 유도한다
3. 다친 척하며 빈틈을 만든다
```

# 13. 무기와 MMA의 관계

## 핵심 원칙

> 무기는 MMA를 대체하지 않는다. 무기는 MMA 위에 얹힌다.

검을 들었다고 전투가 갑자기 화려한 검술 액션이 되면 안 된다.

무기를 들어도 전투의 핵심은 그대로다.

- 거리
- 균형
- 압박
- 호흡
- 붙잡기
- 넘어뜨리기
- 벽 몰기
- 무기 제어

무기는 시스템을 대체하는 것이 아니라, **선택지와 상황을 바꾸는 장비**다.

---

## 무기는 행동 언어를 바꾼다

맨손과 무기 상태는 선택지의 성격을 바꾼다.

### 맨손 상태

```text
상대가 벽 근처로 밀렸다!

1. 벽에 박는다
2. 다리를 건다
3. 태클을 시도한다
```

### 단검 상태

```text
상대가 벽 근처로 밀렸다!

1. 팔을 찌른다
2. 목을 겨눈다
3. 손목을 베고 빠진다
```

### 장검 상태

```text
상대의 자세가 흔들린다!

1. 손등을 친다
2. 검면으로 밀어붙인다
3. 거리를 벌려 다시 겨눈다
```

### 창 상태

```text
상대가 급하게 파고든다!

1. 창대로 밀어낸다
2. 뒤로 빠지며 찌른다
3. 창을 버리고 붙잡는다
```

---

## 무기별 특징

| 무기 | 강점 | 약점 |
|---|---|---|
| 맨손 | 초근접, 붙잡기, 제압 | 거리 싸움 불리 |
| 단검 | 근접 위협, 빠른 공격 | 리치가 짧음 |
| 장검 | 거리 유지, 위협 범위 | 좁은 공간과 클린치에 약함 |
| 창 | 중거리 통제 | 파고들리면 위험 |
| 둔기 | 압박, 균형 붕괴 | 느리고 호흡 소모 큼 |
| 방패 | 밀기, 버티기, 보호 | 공격 전환 느림 |
| 총기 | 원거리 위협 | 초근접에서 제어당할 수 있음 |
| 즉석 무기 | 상황 대응력 | 부서지거나 놓치기 쉬움 |

무기는 단순 공격력 차이가 아니라, 전투의 거리와 위험을 바꾸는 요소다.

---

## 무기전에서도 난투는 유지된다

무기가 있어도 안전해지는 것이 아니다.

예시:

```text
좁은 복도에서 긴 검이 벽에 걸렸다!

1. 검을 짧게 잡고 밀어붙인다
2. 검을 버리고 태클한다
3. 뒤로 빠져 공간을 만든다
```

```text
상대가 당신의 검 든 손목을 붙잡았다!

1. 손목을 비틀어 빼낸다
2. 무릎으로 복부를 친다
3. 검을 놓고 클린치로 전환한다
```

```text
창이 적의 갑옷 틈에 박혔다!

1. 그대로 밀어붙인다
2. 창을 버리고 파고든다
3. 발로 차며 창을 뽑는다
```

이런 구조가 되면 무기전도 스킬전이 아니라 **무기를 든 난투**가 된다.

# 14. 동료 시스템 연계

동료도 자동으로 전투에 참여한다.

## 동료 자동 행동 예시

- 뒤에서 적을 붙잡음
- 대신 맞아줌
- 넘어진 플레이어를 끌어냄
- 적을 벽으로 밀어줌
- 패닉에 빠져 도망감
- 무기를 주워 던짐
- 적의 시선을 끎
- 협공을 시도함

## 유대 수치 영향

동료와의 유대가 높으면 다음 행동이 발생할 수 있다.

- 협동 행동
- 구조 행동
- 연계 공격
- 위험 상황에서 도와줌
- 플레이어의 선택지를 확장함

예시:

```text
동료가 적의 팔을 붙잡았다!

1. 복부를 친다
2. 같이 밀어 넘어뜨린다
3. 동료에게 빠지라고 외친다
```

반대로 동료가 겁을 먹거나 유대가 낮으면 다음 상황도 가능하다.

```text
동료가 뒤로 물러서고 있다!

1. 이름을 부르며 정신 차리게 한다
2. 혼자 밀어붙인다
3. 후퇴를 선택한다
```

전투는 관계 시스템과도 연결된다.

# 15. 군중성과 환경 반응

난투 느낌을 살리려면 주변 반응이 중요하다.

전투는 고립된 1:1 시뮬레이션이 아니라, 주변 세계가 함께 반응하는 난장판이어야 한다.

## 환경 반응 예시

- 직원들이 비명을 지르며 도망감
- 누군가 몰래 촬영함
- 보안팀이 출동함
- 자동문이 닫힘
- 회의실 유리가 깨짐
- 책상이 밀림
- 횃불이 떨어져 불이 붙음
- 구경꾼이 밀려 넘어짐
- 누군가 말리려다가 같이 휘말림

이런 요소는 전투를 단순한 대결이 아니라 살아 있는 상황극으로 만든다.

# 16. 실수 시스템

이 전투에는 반드시 실수가 존재해야 한다.

이 시스템의 맛은 완벽한 액션이 아니라, **멍청할 정도로 현실적인 사고**에서 나온다.

## 실수 예시

- 태클했는데 같이 넘어진다.
- 발이 미끄러진다.
- 너무 세게 밀어서 본인도 벽에 부딪힌다.
- 긴 검이 문틀에 걸린다.
- 의자를 던졌는데 동료 근처로 날아간다.
- 넘어지면서 책상을 엎는다.
- 적을 밀었는데 복사기가 같이 굴러간다.
- 멋있게 피하려다 케이블에 걸린다.

실수는 단순 개그가 아니라 새로운 상황을 생성하는 장치다.

# 17. UI 방향

## 숫자 최소화

UI는 숫자를 최소화한다.

HP 숫자보다 상태바와 문장 중심이 어울린다.

예시:

```text
[적: 일그러진 부장]

균형  █░░░░
압박  ████░
호흡  ██░░░
분노  █████
```

또는 TUI 방식으로 다음처럼 표현할 수 있다.

```text
일그러진 부장
자세: 무너짐
위치: 회의실 벽 근처
호흡: 거칠다
감정: 분노
손: 스테이플러를 쥐고 있음
```

## 선택지 UI

선택지는 짧고 직관적이어야 한다.

```text
부장이 벽에 등을 부딪히며 휘청인다!

1. 어깨로 밀어붙인다
2. 손목을 잡아 스테이플러를 떨어뜨린다
3. 뒤로 빠져 숨을 고른다
```

선택지의 목적은 공격 버튼이 아니라 전투 흐름을 바꾸는 판단이어야 한다.

# 19. 전투 길이와 템포

이 시스템은 전투가 너무 길면 안 된다.

전투가 길어지면 플레이어가 직접 하는 느낌보다 구경하는 느낌이 강해질 수 있다.

따라서 전투는 다음 방향을 지향한다.

- 짧다.
- 상황이 빠르게 바뀐다.
- 개입은 적지만 강하다.
- 선택 결과가 즉시 체감된다.
- 약한 전투는 빠르게 넘긴다.
- 보스전도 3회 이하의 강한 장면으로 구성한다.

전투 하나는 긴 액션 시퀀스가 아니라 **짧은 난투극**이어야 한다.

---

# 20. 주요 리스크와 해결 방향

## 리스크 1. 플레이어가 구경만 하는 느낌

자동 전투 비중이 너무 크면 플레이어가 소외될 수 있다.

해결 방향:

- 개입 횟수는 적게 유지하되 의미 있게 만든다.
- 선택 결과가 즉시 상태 변화로 드러나야 한다.
- 보스전에는 2~3회의 강한 개입 순간을 제공한다.

---

## 리스크 2. 선택지가 너무 자주 떠서 피곤함

개입이 너무 많으면 자동 전투가 아니라 선택지 전투가 된다.

해결 방향:

- 전투당 개입 요구를 0~3회로 제한한다.
- 약한 상대는 0회로 처리한다.
- 평범한 상대는 0~1회로 제한한다.
- 정예몹은 1~2회로 제한한다.
- 보스몹도 최대 3회로 제한한다.

---

## 리스크 3. 선택지가 랜덤처럼 느껴짐

상황과 선택지가 연결되지 않으면 불공정하게 느껴진다.

해결 방향:

- 선택지가 왜 등장했는지 상황 문장으로 설명한다.
- 환경, 거리, 균형, 무기 상태를 명확히 보여준다.
- 사전 단서가 선택지에 반영되게 한다.

---

## 리스크 4. 무기 때문에 평범한 RPG가 됨

검, 창, 마법이 들어오면 스킬 게임으로 변질될 수 있다.

해결 방향:

- 무기도 거리, 균형, 압박 시스템 안에서 작동하게 한다.
- 무기 사용에도 실수와 공간 제약을 넣는다.
- 무기보다 상황을 우선하게 만든다.

---

# 21. 최종 설계 원칙

이 전투 시스템의 최종 원칙은 다음과 같다.

1. 전투의 본체는 자동 난투다.
2. 플레이어는 결정적 순간에만 개입한다.
3. 전투당 개입 요구는 0~3회로 제한한다.
4. 개입 횟수는 전투 시작 전 전투력 비교로 결정한다.
5. 약한 상대는 개입 없이 자동 처리한다.
6. 평범한 상대는 0~1회만 개입한다.
7. 정예몹은 1~2회 개입한다.
8. 보스몹도 최대 3회까지만 개입한다.
9. HP보다 거리, 균형, 압박, 호흡, 무기 제어가 중요하다.
10. 무기는 MMA를 대체하지 않고 MMA 위에 얹힌다.
11. 실수와 환경 반응이 전투를 살아 있게 만든다.
12. 전투는 긴 조작전이 아니라 짧은 전투 드라마다.

# 22. 최종 정리

본 시스템은 다음과 같이 정의할 수 있다.

> 실시간으로 자동 진행되는 MMA 스타일 난투 속에서, 플레이어가 상황을 읽고 제한된 횟수만큼 개입하여 전투의 흐름을 바꾸는 전투 드라마 시스템.

전투의 중심은 공격력이나 스킬 콤보가 아니라 **흐름**이다.

플레이어는 매 순간 공격 버튼을 누르는 사람이 아니라, 중요한 순간에만 판단을 내려 싸움의 방향을 바꾸는 사람이다.

약한 적은 자동으로 처리되고, 평범한 적은 가볍게 지나가며, 정예와 보스는 1~3회의 강한 개입 순간을 통해 기억에 남는 전투 장면을 만든다.

최종적으로 이 전투의 고유한 맛은 다음 문장으로 요약된다.

> 검을 든 기사도, 마법사도, 회사원도, 괴물도 결국 가까이 붙으면 밀치고, 붙잡고, 넘어지고, 숨차고, 당황한다.
