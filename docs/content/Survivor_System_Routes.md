# 생존자/시스템 제압·설득 루트

이 문서는 Phase 9의 남은 큰 덩어리인 “생존자 또는 시스템 제압/설득 설계”를 현재 런타임 hook에 맞춰 public-safe 형태로 정리한다.
새 실제 인물, 실제 회사 구조, 실제 보안 절차를 만들지 않고, 이미 구현된 사내망·방송·서버실·보안실 루트를 확장 가능한 설계 단위로 묶는다.

## 설계 원칙

- public-safe: 공개 저장소에는 실제 직원, 실제 부서, 실제 시스템 이름을 넣지 않는다.
- 생존자는 “실제 사람”보다 격리 프로토콜 안에 남은 업무 잔상, 녹음, 지연 메시지로 먼저 표현한다.
- 제압은 폭력이 아니라 권한, 결재선, 방송 채널, 출입 규칙을 덮어쓰는 행위다.
- 설득은 시스템이 중요하게 여기는 업무 정합성을 이용해 생존 조건을 승인받는 행위다.

## 생존자 설득 루트

목표 감정: 혼자가 아니라는 안도감과, 그 상대가 정말 사람인지 모른다는 불안을 동시에 준다.

### 현재 hook

| 단계 | hook | 의미 |
|---|---|---|
| 접촉 | `internal_channel_key` | 서버실 앞 무전기에서 제한된 주파수 확보 |
| 방송 통제 | `server_room_broadcast_controlled` | 사내 방송 채널 장악 |
| 진실 단서 | `server_log_fragment` | 지연된 로그에서 생존자/프로토콜 흔적 확인 |
| 회의 패턴 | `meeting_pattern_noticed` | 존재하지 않는 회의의 반복 규칙 인지 |

### 후보 흐름

1. 무전기 또는 사내 방송에서 “아직 퇴근 처리되지 않은 사용자”의 음성이 들린다.
2. 플레이어는 방송 채널을 끊지 않고 안정화한다.
3. 회의록 패턴과 서버 로그 조각을 맞춰 상대가 생존자인지, 업무 잔상인지 판단한다.
4. 설득 성공 시 시스템은 플레이어의 생존을 “필수 참석자 유지”로 승인한다.
5. 실패 시 상대 목소리는 사내 방송의 안내 멘트로 흡수된다.

## 시스템 제압 루트

목표 감정: 탈출하지 못했지만, 회사 규칙을 플레이어가 다시 쓴다는 블랙코미디 정복감.

### 현재 hook

| 단계 | hook | 의미 |
|---|---|---|
| 우회권한 | `security_override_badge` | 보안실에서 서버실 우회권한 확보 |
| 서버실 진입 | `security_override_used` | 우회권한으로 서버실 문 예외 처리 |
| 관리자 권한 | `network_admin_claimed` | 관리자 콘솔에서 격리 규칙 덮어쓰기 |
| 보안 격리 | `security_lockdown_claimed` | 출입 로그와 격리 규칙 동시 잠금 |
| 대표 승인 | `executive_approval_claimed` | 대표실 결재 콘솔 장악 |
| 정책 변경 | `company_policy_overwritten` | 결재선을 플레이어에게 귀속 |

### 후보 흐름

1. 보안실에서 출입 기록의 어긋난 층수를 서버실 예외 권한으로 접는다.
2. 서버실 콘솔에서 격리 규칙의 소유자를 플레이어로 바꾼다.
3. 사내 방송 또는 대표실 결재 콘솔을 통해 새 규칙을 공지한다.
4. 시스템은 플레이어를 감시 대상이 아니라 예외 승인자로 취급한다.
5. 엔딩은 `conquest_network_admin`, `conquest_security_lockdown`, `conquest_broadcast_channel`, `conquest_executive_approval` 중 조건에 따라 갈린다.

## 설득과 제압의 차이

| 축 | 생존자 설득 루트 | 시스템 제압 루트 |
|---|---|---|
| 핵심 자원 | 정신력, 배터리 | 배터리, 권한 아이템 |
| 주요 장소 | 서버실 앞, 회의실, 보안실 | 보안실, 서버실, 대표실 |
| 실패 감정 | 목소리가 시스템 안내로 흡수됨 | 회사가 플레이어를 관리자 대신 장애로 분류함 |
| 성공 감정 | 생존 조건을 승인받음 | 퇴근 규칙을 직접 작성함 |

## 현재 구현과의 경계

이 문서는 설계 완료 상태를 표시하지만, 새 런타임 인카운터를 추가하지 않는다.
현재 Phase 9 체크리스트의 “설계” 항목은 완료하고, 실제 신규 선택지/엔딩 추가는 다음 콘텐츠 확장 slice에서 다룬다.
